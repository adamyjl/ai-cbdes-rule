请为智能驾驶控制模块（`control`）开发或修复一个功能，用于计算二维平面上两点 $(x_1, y_1)$ 和 $(x_2, y_2)$ 之间的欧几里得距离，并计算这两点的中心点坐标 $(x_c, y_c)$。

**目标**：
实现一个函数，接收四个 double 类型参数（x1, x2, y1, y2），返回一个包含距离（distance）、中心点 X 坐标（xc）和中心点 Y 坐标（yc）的结构体或元组。

**输入输出约束**：
- 输入参数：`double x1`, `double x2`, `double y1`, `double y2`
- 输出结果：`double distance`（欧几里得距离）, `double xc`（中心点 X 坐标）, `double yc`（中心点 Y 坐标）

**边界条件**：
- 输入无效时（如 NaN 或 Inf），输出也应返回无效。

**验收标准**：
- 正确调用现有的 `get2Dis` 函数计算距离。
- 正确计算中心点坐标：$x_c = \frac{x_1 + x_2}{2}$, $y_c = \frac{y_1 + y_2}{2}$。
- 实现输入有效性检查逻辑。
- 编写单元测试用例，覆盖正常输入、边界值及无效输入情况。

**建议修改点**：
- 复用或调用现有的 `get2Dis` 函数计算两点间距离。
- 将上述两个逻辑整合，封装为一个新的函数或修改现有函数以返回结构体/元组包含距离和中心点。

**引用相关函数名/路径**：
- `get2Dis` 函数位于 `Control/LQRController/dependence/controlController.hpp`，其实现为 `static inline double get2Dis(double x1, double x2, double y1, double y2) { return sqrt(pow((x1 - x2), 2) + pow((y1 - y2), 2)); }`。
